Quản Lý Asynchronous trong JavaScript

Kiến thức về quản lý các tác vụ bất đồng bộ trong JavaScript, bao gồm:

  • Callback: Hàm gọi lại
  • Promise: Kiểu dữ liệu để làm việc với các tác vụ bất đồng bộ
  • Async/Await: Cú pháp để xử lý các hàm bất đồng bộ một cách dễ dàng

1. Callback

Callback là một hàm được truyền như một đối số cho một hàm khác và sẽ được gọi lại sau khi một tác vụ hoàn tất.


function fetchData(callback) {
    setTimeout(() => {
        const data = "Dữ liệu đã được lấy!";
        callback(data);
    }, 2000);
}

fetchData((result) => {
    console.log(result); // In ra: Dữ liệu đã được lấy!
});

2. Promise

Promise là một đối tượng đại diện cho một giá trị có thể có trong tương lai. Promise có thể ở trong một trong ba trạng thái: pending (đang chờ), fulfilled (đã hoàn thành) hoặc rejected (đã bị từ chối).

2.1. Tạo Promise


let myPromise = new Promise((resolve, reject) => {
    const success = true; // Giả sử tác vụ thành công
    if (success) {
        resolve("Tác vụ thành công!");
    } else {
        reject("Tác vụ thất bại!");
    }
});

// Sử dụng Promise
myPromise
    .then(result => console.log(result)) // In ra: Tác vụ thành công!
    .catch(error => console.error(error));

3. Async/Await

Async/Await là cú pháp được giới thiệu trong ES2017 giúp xử lý các hàm bất đồng bộ một cách dễ dàng hơn. Hàm được đánh dấu bằng từ khóa async và bên trong hàm có thể sử dụng await để chờ kết quả từ một Promise.

3.1. Sử dụng Async/Await


async function fetchData() {
    try {
        let result = await myPromise; // Chờ cho Promise hoàn thành
        console.log(result); // In ra: Tác vụ thành công!
    } catch (error) {
        console.error(error);
    }
}

fetchData();
