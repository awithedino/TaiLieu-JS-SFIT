Kiểu dữ liệu trong JavaScript

JavaScript không tự động nhận diện kiểu của biến theo cách mà các ngôn ngữ có kiểu dữ liệu tĩnh (như C, C++ hay Java) thường làm. Thay vào đó, JavaScript sử dụng hệ thống kiểu dữ liệu động (dynamic typing), nghĩa là một biến có thể chứa bất kỳ kiểu dữ liệu nào, và kiểu dữ liệu có thể thay đổi trong quá trình thực thi chương trình mà không cần khai báo rõ ràng trước.

Cách hoạt động của hệ thống kiểu động:

  1. Tự động suy luận kiểu dữ liệu: Khi bạn gán một giá trị cho biến, JavaScript sẽ tự động nhận diện kiểu dữ liệu của giá trị được gán và lưu trữ nó trong biến. Bạn không cần khai báo kiểu dữ liệu của biến khi khai báo. 
    let x = 42;        // Biến x có kiểu "number"
x = "Hello";       // Bây giờ, x là kiểu "string"
x = [1, 2, 3];     // Bây giờ, x là kiểu "array" (một đối tượng trong JavaScript)
  2. Thay đổi kiểu dữ liệu của biến: Do tính động của JavaScript, bạn có thể gán giá trị kiểu dữ liệu khác nhau cho cùng một biến trong suốt quá trình thực thi. JavaScript sẽ tự động thay đổi kiểu dữ liệu của biến dựa trên giá trị mới.

Một số ví dụ:

let y = 100;          // y có kiểu "number"
console.log(typeof y); // Kết quả: "number"

y = "JavaScript";     // y giờ có kiểu "string"
console.log(typeof y); // Kết quả: "string"

y = [1, 2, 3];        // y giờ là một mảng
console.log(typeof y); // Kết quả: "object" (Trong JavaScript, mảng là một dạng đối tượng)

1. Kiểu dữ liệu cơ bản

JavaScript hỗ trợ một số kiểu dữ liệu cơ bản, bao gồm:

1.1. String

  • Mô tả: Là kiểu dữ liệu để lưu trữ chuỗi ký tự.
  • Cú pháp: Có thể sử dụng dấu nháy đơn ('), dấu nháy đôi ("), hoặc dấu nháy ba ( `` ).
  • Ví dụ: 
    let name = 'Mai Thanh Dat';
let greeting = "Chào mừng bạn đến với JavaScript!";
let message = `Xin chào, ${name}!`;  // Template literals
console.log(message);  // Xin chào, Mai Thanh Dat!

1.2. Number

  • Mô tả: Là kiểu dữ liệu để lưu trữ số, bao gồm cả số nguyên và số thực.
  • Ví dụ: 
    let age = 25;
let height = 1.75;
let total = age + height;  // 26.75
console.log(total);

1.3. Boolean

  • Mô tả: Là kiểu dữ liệu để lưu trữ giá trị đúng hoặc sai (true/false).
  • Ví dụ: 
    let isActive = true;
let isAdult = age >= 18;  // true
console.log(isAdult);

1.4. Undefined

  • Mô tả: Là kiểu dữ liệu chỉ ra rằng biến đã được khai báo nhưng chưa có giá trị.
  • Ví dụ: 
    let x;
console.log(x);  // undefined

1.5. Null

  • Mô tả: Là kiểu dữ liệu để chỉ định một giá trị "không" hoặc "trống".
  • Ví dụ: 
    let y = null;
console.log(y);  // null

1.6. Symbol

  • Mô tả: Là kiểu dữ liệu mới trong ECMAScript 6, được sử dụng để tạo ra các giá trị duy nhất không thể so sánh.
  • Ví dụ: 
    let sym = Symbol('description');
console.log(sym);  // Symbol(description)

2. Kiểu dữ liệu phức tạp

2.1. Object

  • Mô tả: Là kiểu dữ liệu để lưu trữ tập hợp các cặp key-value.
  • Cú pháp: 
    let person = {
    name: 'Mai Thanh Dat',
    age: 25,
    isActive: true
};
console.log(person.name);  // Mai Thanh Dat

2.2. Array

  • Mô tả: Là kiểu dữ liệu để lưu trữ danh sách các giá trị.
  • Cú pháp: 
    let fruits = ['Apple', 'Banana', 'Cherry'];
console.log(fruits[0]);  // Apple

3. Kiểm tra kiểu dữ liệu

Để kiểm tra kiểu dữ liệu của một biến, bạn có thể sử dụng toán tử typeof:

Ví dụ:

let x = 10;
console.log(typeof x);  // "number"

let y = 'Hello';
console.log(typeof y);  // "string"

let z = true;
console.log(typeof z);  // "boolean"

let obj = { name: 'Mai' };
console.log(typeof obj);  // "object"

let arr = [1, 2, 3];
console.log(typeof arr);  // "object" (arrays are also objects in JS)

let u;
console.log(typeof u);  // "undefined"

let n = null;
console.log(typeof n);  // "object" (this is a known bug in JS)
